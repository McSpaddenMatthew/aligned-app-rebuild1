Quick instructions

Paste your Supabase URL and anon key into the script constants at the bottom of this file. Then run the quick checks to confirm connectivity.

  • Click “Create test user” to store a fake profile.
  • Use “List messages” to see demo inbox data.
  • Review the timeline to understand data flow.

Onboarding

Creates a new workspace, seeds a few inbox messages, and links actions to the user record so the dashboard is ready on first load.

Context-aware AI

We enrich each message with sender reputation, historical cadence, and existing tasks. The AI can draft the best follow-up without guesswork.

Safety rails

Templates are built from typed actions. Users approve before sending, and we log every call with rationale and confidence.

Handoff-ready

Export a summary packet with every decision, open thread, and next action. Human teammates can review in seconds.

How data flows

01
Auth / session

Use Supabase Auth to create or sign in a user. Store the session token for subsequent reads on their workspace.

02
Seed inbox

Insert demo messages into messages and actions tables to populate the interface immediately.

03
Generate guidance

Call the hosted model with the synced context and render suggestions the user can approve or edit.

04
Ship work

Send the reply via your preferred channel and log the execution for analytics and undo capability.
